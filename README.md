# cashback_mcc_tool
Автономное веб-приложение в одном HTML-файле для ведения базы банковских карт, их категорий кэшбэка по MCC-кодам и автоматического формирования ежемесячного документа «где какой кэшбэк выгоднее»
# Руководство пользователя — «Кэшбэк по MCC» (v161)

> Что это: автономное веб-приложение в одном HTML-файле для ведения базы банковских карт,
> их категорий кэшбэка по MCC-кодам и автоматического формирования ежемесячного документа
> «где какой кэшбэк выгоднее». Работает локально в браузере, без установки и без сервера.
> Данные хранятся в `localStorage`; для переноса и резервных копий используется JSON.

## 1. Быстрый старт

1. Откройте файл `cashback_mcc_tool_v161.html` в браузере.
2. Если база ещё не заполнена под вас — работайте во вкладке **Редактор базы**.
3. Добавьте свои продукты и категории либо импортируйте JSON через кнопку **Импорт**.
4. В объединённой вкладке **Редактор базы** отметьте активные категории чекбоксами и при необходимости скорректируйте проценты только на текущий месяц.
5. Укажите месяц и MCC-коды из своей статистики в верхнем блоке.
6. Нажмите **Сформировать документ**.
7. При желании нажмите **Применить как документ на месяц**, чтобы закрепить результат на отдельной вкладке.

> Важно: данные сохраняются локально в браузере. Перед крупными правками делайте **Экспорт JSON**.

## 2. Что хранится в приложении

### Банковский продукт
У продукта есть:
- короткое имя для бейджа;
- банк и полное название;
- цвета бейджа и жирность;
- общие примечания;
- MCC-исключения для REST;
- условия активации и лимиты;
- ссылки на документы банка;
- форма кэшбэка (рубли/баллы) и сроки начисления;
- список категорий.

### Категория
У категории есть:
- внутреннее название;
- процент кэшбэка;
- MCC-коды и диапазоны;
- примечания (показываются в подсказке при наведении на иконку вопроса);
- краткие ограничения для вывода в документ;
- признак REST («Все остальные»);
- признак дополнительной суммируемой категории;
- пометки экосистем **Яндекс** и **Сбер**.

### Что такое REST
REST — это ставка «на всё остальное», если MCC не попал в выбранные повышенные категории.
REST **независим у каждого продукта**: свои исключения, свой процент, своя категория.

## 3. Шапка приложения и вкладки

Верхняя панель содержит:

## 4. Вкладка «Редактор базы»

Здесь вы одновременно редактируете базу продуктов и отмечаете активные категории для формирования документа.

### Верхняя часть страницы
В верхнем белом блоке расположены:
- поле **Месяц**;
- поле **MCC-коды из вашей статистики**;
- поле **Ваши категории** (необязательно);
- радио-кнопки **Какие MCC включать в результат**;
- слайдер **Минимальная ширина карточки продукта**.

Подробнее об этих полях — в разделе [Верхний блок параметров месяца](#5-верхний-блок-параметров-месяца).

### Кнопки действий (верх и низ)
- **Добавить банковский продукт** — создать новую карточку.
- **Очистить базу** — удалить все продукты сразу.
- **Сравнить категории** — перейти к сравнению MCC по банкам.
- **Сформировать документ** — сгенерировать документ на выбранный месяц.
- **Сохранить базу (localStorage)** — принудительное сохранение.

Все эти кнопки продублированы и вверху страницы (под заголовком), и внизу (после сетки карточек).

### Сетка карточек продуктов
Карточки продуктов могут располагаться в адаптивной сетке в несколько колонок. Каждая карточка содержит полную информацию о продукте и его категориях.

### Минимальная ширина карточки продукта (слайдер)
Регулятор-слайдер, расположенный рядом с селектором «Какие MCC включать в результат». Позволяет настраивать минимальную ширину карточек продуктов в реальном времени от 80% до 200% (от 240px до 600px). Настройка автоматически сохраняется в памяти браузера.

## 5. Верхний блок параметров месяца

### Месяц
Выпадающий список из 12 месяцев. Используется в заголовке сформированного документа и в названии вкладки «Кэшбэк на <месяц>».

### MCC-коды из вашей статистики
Текстовое поле для ввода MCC-кодов из вашей банковской статистики.
- Формат: по одному на строку или через запятую.
- Можно указывать диапазоны (например, `5812-5815`).
- При первом открытии подставляется стандартный набор частых MCC.

### Ваши категории (необязательно)
Поле для задания своих названий категорий и привязки примеров торговых точек к MCC.

**Формат:**
```text
Название моей категории: MCC1 (пример1, пример2), MCC2 (пример3)
```

Пример:
```text
Супермаркеты: 5411 (Пятёрочка, Перекрёсток), 5499 (Магнит)
Кафе и фастфуд: 5812 (Кафе у дома), 5814 (Вкусно и точка, Rostic's)
```

Если поле заполнено — в документе будут использоваться ваши названия категорий вместо отдельных MCC-блоков.

### Какие MCC включать в результат
- **Только указанные MCC** — в документ попадут только коды из вашей статистики.
- **Указанные + все точные MCC из выбранных категорий** — дополняет ваши MCC точными кодами из отмеченных категорий базы.
- **Комбинация (рекомендуется)** — оптимальный режим, объединяющий оба подхода.

## 6. Карточка продукта

### Что видно на карточке
- бейдж с цветами продукта;
- банк в скобках;
- полное название;
- количество категорий;
- краткие лимиты;
- форма кэшбэка и сроки начисления;
- примечания;
- **список категорий с чекбоксами и полями процента**;
- ссылки на документы.

### Действия с продуктом
- **↑ / ↓** — поменять порядок продукта.
- **✎** (иконка карандаша) — открыть полное редактирование продукта.
- **🗑** (иконка корзины) — удалить продукт.
- Перетаскивание мышью тоже работает (drag & drop).
- Клик по карточке (вне кнопок и интерактивных элементов) открывает редактирование.

### Что есть в редактировании продукта
- название продукта;
- цвет текста, цвет фона;
- живой предпросмотр бейджа;
- банк и полное название;
- примечания;
- условия и лимиты;
- MCC-исключения;
- ссылки на документы;
- форма кэшбэка и сроки начисления;
- список категорий с кнопками добавления/редактирования/удаления;
- кнопки «Сохранить изменения», «Отмена» и «Удалить продукт».

### Категории в карточке

Каждая категория в списке отображается в виде строки с элементами:

#### Чекбокс
Слева от названия категории. По умолчанию **все категории отмечены**. Снятие чекбокса исключает категорию из расчёта при формировании документа. Если в настройках категории включена опция «Активна всегда», чекбокс отображается серым (неактивным) и категория всегда участвует в расчёте.

#### Название категории
Если у категории нет явных MCC-кодов, название выделяется цветом бейджа продукта.

#### Поле процента (%)
Справа от названия. Показывает **эффективный процент** на текущий месяц:
- если вы не меняли процент — отображается базовый процент из базы;
- если вы вручную изменили процент — отображается ваше значение.

Ручное изменение процента сохраняется автоматически и действует только до следующего изменения.

#### Иконка вопроса (примечания)
Если у категории есть примечания (`notes`), справа от названия появляется иконка **?** в кружке:
- **При наведении** курсора на иконку с задержкой в 200 мс появляется всплывающая подсказка с текстом примечания.
- Подсказка исчезает через 1 секунду после ухода курсора.
- **При клике или тапе** на иконку подсказка появляется/исчезает (удобно на мобильных устройствах).

#### Бейджи
- **REST** — базовая категория «Все остальные».
- **+доп** — суммируемая категория.
- **Я✓ / Я✗ / С✓ / С✗** — экосистемные пометки (не влияют на расчёт).

#### Кнопка «Снять все»
Находится справа от заголовка «Категории:» в каждой карточке продукта. Одним нажатием снимает все чекбоксы у данного продукта.

#### Кнопки редактирования категории
- **✎** — редактировать категорию (открывает модальное окно).
- **🗑** — удалить категорию.

## 7. Карточка категории

Категория редактируется в модальном окне.

Поля:
- **Внутреннее название** — служебное имя категории.
- **Кэшбэк, %** — число от 0 до 100, дробные значения допустимы.
- **Это REST («Все остальные»)** — базовая ставка продукта.
- **Дополнительная (суммируется)** — процент прибавляется к основной повышенной категории.
- **Активна всегда** — категория кэшбэка действует независимо от выбора (всегда выбрана, не сбрасывается кнопкой «Снять все»).
- **MCC-коды** — точные коды и диапазоны.
- **Примечания к категории** — подробные комментарии (видны в tooltip при наведении на иконку вопроса в карточке продукта).
- **Ограничения этой категории** — короткая подпись для документа.
- **Поддержка экосистем (Яндекс / Сбер)** — только информационные флаги.

### Нормализация MCC при сохранении
Приложение автоматически:
- удаляет дубликаты;
- удаляет точные коды, которые уже покрыты диапазоном;
- сворачивает 3 и более подряд идущих кода в диапазон;
- сортирует список.

Если нормализация реально что-то убирает, приложение показывает подтверждение.

## 8. Сформированный документ

После нажатия **Сформировать документ** приложение строит документ по текущему выбору и автоматически переключается на вкладку **Документ** (новая вкладка).

### Что внутри
Для каждого MCC:
- слева код MCC;
- если есть — пользовательское название категории;
- если есть — примеры торговых точек в скобках;
- ниже — список продуктов с процентами кэшбэка, отсортированный по убыванию.

### Важные детали вывода
- ограничения категории показываются рядом с продуктом обычным текстом, без цветной заливки;
- цветные бейджи продуктов визуально разделяются, чтобы не слипаться;
- по клику на продукт можно открыть карточку с деталями;
- по клику на MCC можно открыть пользовательское примечание из поля «Ваши категории».

### Дополнительные секции
- **Выбранные категории без явных MCC-кодов** — для банковских сервисных категорий без списка MCC;
- **Примечания и исключения** — краткая памятка по использованным продуктам.

### Доступные действия
- **Применить как документ на месяц** — сохраняет снимок на вкладку «Кэшбэк на <месяц>»;
- **Текст** — копирование plain-text версии;
- **Markdown** — копирование markdown-представления;
- скачивание в **.txt**, **.md**, **.html**.

### Экспорт `.html`
HTML-экспорт самодостаточный: документ открывается отдельно от приложения и сохраняет читаемую разметку, включая визуально разделённые бейджи.

## 9. Вкладка «Сравнение MCC»

Это отдельный режим анализа базы по вашим MCC.

### Как запустить
В объединённой вкладке **Редактор базы** нажмите **Сравнить категории**.

### Что показывает сравнение
Для каждого MCC приложение выводит:
- пользовательское название категории, если оно задано;
- примеры торговых точек, если они указаны;
- список банков и категорий, где этот MCC встречается;
- исключения REST.

### Режимы просмотра
- **Список** — обычный текстовый список по MCC.
- **Таблица** — матрица MCC × банки.

### Цвета в таблице
- **зелёный** — MCC найден только в категориях кэшбэка;
- **оранжевый** — MCC есть и в категориях, и в исключениях;
- **красный** — MCC есть только в исключениях.

### Экспорт сравнения
На вкладке сравнения есть отдельные кнопки:
- **.html** — цветная таблица;
- **.txt** — текстовая версия.

## 11. Экспорт, импорт и сброс

### Экспорт JSON
Сохраняет не только базу, но и состояние селектора:
- `products`;
- `selections` — отмеченные категории;
- `overrides` — ручные проценты;
- `customCategories` — текст поля «Ваши категории»;
- `lastMonth` — последний выбранный месяц.

### Импорт JSON
После выбора файла приложение анализирует отличия и показывает модальное окно.

Варианты действий:
- **Применить все**;
- **Применить выбранное**;
- **Полная замена базы**.

### Полная замена базы
Эта кнопка:
- полностью заменяет базу продуктов данными из файла;
- восстанавливает чекбоксы, ручные проценты, «Ваши категории» и месяц, если они есть в файле;
- если импортирован старый JSON без состояния селектора — текущие галочки и override-проценты сбрасываются.

### Сброс
Кнопка **Сброс**:
- либо возвращает ваши сохранённые умолчания;
- либо загружает встроенную демо-базу.

### Сохранить как умолчания
Сохраняет текущие:
- базу;
- выбранные категории;
- ручные проценты.

## 12. Как считается кэшбэк

Для каждой пары **продукт + MCC** приложение делает так:

1. Среди отмеченных (чекбоксом) категорий ищет все повышенные категории, куда попадает MCC.
2. Если обычных повышенных категорий несколько — берёт **максимальный** процент.
3. Если есть дополнительные (`isAdditive`) категории — их проценты **суммируются**.
4. Если повышенных категорий нет, но выбрана REST-категория:
   - при отсутствии исключения берётся REST-процент;
   - если MCC есть в исключениях — кэшбэк по REST не начисляется.
5. Результат округляется до десятых.

Формула в общем виде:

```text
итог = max(обычные повышенные) + сумма(дополнительные)
```

REST используется только если повышенные категории по этому MCC не сработали.

### Ручные проценты (override)
Если вы изменили процент в поле рядом с категорией, это значение подменяет базовый процент **только для текущего месяца**. Расчёт документа использует именно это значение.

## 13. Полезные замечания и FAQ

### Где хранятся данные?
В `localStorage` браузера на текущем устройстве.

### Можно ли работать без интернета?
Да. Это локальный HTML-файл. Внешние стили могут деградировать, но логика приложения остаётся рабочей.

### Можно ли использовать диапазоны MCC?
Да, например `5411-5499`.

### Почему приложение иногда спрашивает подтверждение при сохранении MCC?
Потому что нормализация может убрать дубли или коды, уже покрытые диапазонами.

### Экосистемные значки влияют на расчёт?
Нет. Они только показывают справочную информацию.

### Почему категория без MCC попадает в отдельный блок?
Потому что она не может быть привязана к конкретному коду, но всё равно была выбрана.

### Можно ли импортировать старый JSON?
Да. Приложение поддерживает миграцию старых данных.

### Что делать перед крупной чисткой или массовым импортом?
Сначала сделать **Экспорт JSON**.
